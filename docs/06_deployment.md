# 06. Project Deployment Effort and Use Case

## Deployment Overview

This project is deployed and presented through a GitHub repository that contains the full documentation, data structure, simulation files, dashboard resources, financial analysis materials, and app prototype link.

The goal of deployment is to make the project easy to review, navigate, and understand by the supervisor, examiners, and potential business users.

---

## Repository-Based Deployment

The GitHub repository is the main deployment platform for the graduation project. It allows the project to be organized in a professional structure and supports version control.

The repository includes:

- Main README file
- Markdown documentation files
- Data folders
- Python notebooks or scripts
- PowerPoint presentation
- Simio model file
- Dashboard and app prototype materials
- Setup instructions
- Evaluation criteria

---

## Business User Use Case

A plant manager or decision-maker can use this project to understand whether the BioFuel waste tyre pyrolysis system is worth developing further.

The user can review:

1. The simulation results to understand system flow and bottlenecks.
2. The economic analysis to understand investment and profitability.
3. The environmental assessment to understand sustainability value.
4. The app prototype to understand how digital monitoring can support daily operations.

---

## Operational Use Case

In a real plant setting, the digital monitoring app concept could help operators monitor:

- Machine status
- Reactor batch status
- Material levels
- Temperature and humidity
- Alerts and warnings
- Reports
- Output indicators

The most important monitoring focus is the reactor, because it is the main bottleneck and has the strongest effect on production and profit.

---

## Deployment Steps

The project deployment process includes:

1. Organize project files into the required GitHub structure.
2. Write project documentation in Markdown.
3. Upload the Simio model file to the `models/` folder.
4. Upload the presentation and dashboard visuals to the `dashboards/` folder.
5. Store raw and processed data files in the `data/` folder.
6. Add notebooks or scripts in the `notebooks/` and `src/` folders.
7. Add setup instructions in `docs/SETUP.md`.
8. Link all major files through the main `README.md`.
9. Test all GitHub links to make sure they open correctly.

## Limitations of Deployment

The current deployment is a prototype and academic project submission. It does not include live industrial data connections or real-time machine integration.

For real-world deployment, the following would be required:

- Live sensors
- Industrial data connection
- Real-time database
- User authentication
- Safety monitoring system
- Maintenance module
- Automated alerts
- Cloud hosting
- Integration with plant equipment

---

## Conclusion

The deployment approach makes the project easy to access, review, and understand. The GitHub repository works as the main project hub, while the app prototype demonstrates how digital monitoring can support the BioFuel pyrolysis system in the future.
